TAZ OFFSET FIX — VALIDATION REPORT
=====================================

PROBLEM IDENTIFIED (Professor's Feedback)
------------------------------------------
Original code searched for POIs in HOME TAZ for ALL activity types.
This ignores that real people travel to different zones for activities.

ROOT CAUSE
----------
The code copied agent's absolute activity_taz directly.
Since each city area has different TAZ numbers, this placed
synthetic agents' activities in completely wrong zones.

BEFORE FIX — WRONG BEHAVIOUR
-------------------------------
Real agent:    home_taz=5180  →  shop_taz=5247  (offset: +67, ~5km away)
Synthetic:     home_taz=3415  →  shop_taz=5247  (WRONG: 35km away!)

Result: A synthetic person living in southwest Orange County
was sent to shop at POIs in the north Orlando suburb where the
MATCHED AGENT lives — not near their own home.

AFTER FIX — CORRECT BEHAVIOUR
--------------------------------
Code: activity_taz = synthetic_home_taz + (agent_activity_taz - agent_home_taz)
         = 3415 + (5247 - 5180)
         = 3415 + 67 = 3482

Result: Synthetic person now searches for a POI ~5km from their
own home, matching the real agent's relative travel pattern. ✓
